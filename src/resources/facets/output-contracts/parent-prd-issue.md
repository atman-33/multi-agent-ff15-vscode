## Format

````markdown
---
prd_key: <same key as requirements brief>
issue_number: <github issue number>
issue_url: <absolute github issue url>
issue_title: <github issue title>
issue_action: <create or update>
---
````

## Rules

- `prd_key` must match the `PRD Key` in `requirements-brief.md` and `prd-draft.md`
- Record `issue_number` as numbers only
- `issue_url` must be the absolute URL of the GitHub issue
- `issue_title` must match the published GitHub issue title
- `issue_action` must be either `create` or `update`
